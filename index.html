sed流编辑器

发表于 2020-04-16 | 分类于 shell
  • 作用

sed是Stream Editor的缩写，即流编辑器，用来处理文件

  • 处理流程

1.sed首先逐行读取文件内容，将其保存在sed的模式空间，即缓冲区
2.根据需求处理模式空间的内容后，打印到屏幕上

  • sed常见的语法格式
    1. 命令行格式

语法：

sed [option] '处理动作' 文件名

常用选项

-e 进行多次（多项）编辑
-n 取消默认输出（即不自动打印模式空间的内容） -n ‘p’ 只打印一遍内容
-r 使用扩展正则表达式
-f 指定sed脚本的文件名
-i 修改源文件

常见处理动作

‘p’ 打印
‘i’ 在指定行之前插入内容 （不要加-n会删除其他行 和p修改行会重复） 类似vim的大写O
‘a’ 在指定行之后插入内容 类似vim的小写o
‘c’ 替换指定行的所有内容
‘d’ 删除指定行

  • 示例
    1. 查

      sed [opt] ‘定位+命令’ 文件

    sed '' 1.txt    #什么都不操作，结果会把模式空间的内容输出到屏幕
sed 'p' 1.txt   #执行打印操作，重复输出内容，一个是'p'的结果，一个是默认输出的结果
sed -n '1p' 1.txt    #输出一遍1.txt的第一行内容
sed -n '1，4p' 1.txt    #输出一遍1.txt的第一到四行内容
sed -n '$p' 1.txt   #输出一遍1.txt的最后一行内容
> 2. 增
>> sed '定位+a新增内容' 文件名
>> sed '定位'

sed 'ihelloworld' 1.txt   在每一行的前面插入helloworld
sed '2ihelloworld' 2.txt  在第二行前面插入
sed '2,3ihelloworld' 1.txt 在第二第三行前插入

3.修改

sed ‘c’ 替换整行内容

sed '$chelloworld' 1.txt    把最后一行替换为helloworld
sed '/^admin/chelloworld'   把以admin开头的一行替换为。。
sed '1,5chelloworld'    把1到5行的内容替换成一行helloworld

4.删除

sed ‘d’ 删除整行

sed 'd' 1.txt   删除全部行
sed '1,4d' 1.txt    删除1到4行
sed '$d' 1.txt 删除最后一行
sed '/[0-9]/d' 1.txt 删除数字所在行

5.对文件进行搜索替换操作（/分隔符可以自定义）
语法：
sed ‘s/搜索的内容/替换的内容/动作’ 需要处理的文件
s表示搜索，默认全局搜索，/是分隔符，可以自己定义；动作一般可以是p打印或g全局替换

sed -n 's/huyun/jiang/p' 1.txt  把第一个匹配到的huyun替换成jiang，并打印这一行
sed -n '10s/a/b/p' 1.txt 第十行
sed -n '1,5/^/#/gp' 1.txt   注释第一到五行
  1. 其他命令
    sed ‘3r /etc/file1.txt’ file2.txt r 读取file1.txt文件内容到file2.txt的第三行
    sed ‘1,5w file1.txt’ file2.txt w 保存file2.txt下第一到五行的内容到file1.txt
    sed ‘s/^sea/#&/p’ file1.txt &保存查找的串以便引用，把sea开头的行注释
    sed -ne ‘/root/=;/root/p’ 打印root匹配的行号和该行内容
    sed ‘1,5!’ 匹配第五行后的内容
    sed ‘/shutdown/q’ 匹配到shutdown后退出

6.结合正则使用

sed 选项 ‘sed命令或者正则表达式或者地址定位’ 文件名
地址用于决定对哪一行进行编辑，可以是数字、正则表达式，不输入地址则匹配所有行

/key/   查询包含关键字的行      sed -n '/root/p' 1.txt
/key1/,/key2/   匹配包含两个关键字之前的行      sed -n '/^admin/,/mysql/p' 1.txt
/key/,num   匹配包含关键字所在行和第num行之间的行，包含关键字所在行 sed -n '/admin/,4p' 1.txt
num,/key/   num行到与关键字匹配行之间的行
x,y!    不包含x到y行
/key/!  不包括关键字的行

7.sed的脚本模式

# sed -f script.sh file     使用命令处理文件

脚本格式如下：
#!/bin/sed -f
command1
command2

名词解释

  • 元字符
    具有特殊含义的字符，如. * ?等
  • 前导字符
    位于元字符前面的一个字符

第一类正则表达式

  • 常用元字符
    ```
    . 匹配除了换行符以外的其他任意单个字符
  • 前导符出现0次或连续多次
    .* 任意长度字符
    ^ 行首
    $ 行尾
    ^$ 空行
    [] 匹配括号中任意单个字符或一组单个字符 [abc] 匹配a或b或c
    [^] 匹配不包含括号中任意单个字符或一组单个字符
    ^[] 匹配以括号中任意单个字符或一组单个字符开头
    ^[^] 匹配不以括号中任意单个字符或一组单个字符开头
    ```
  • 其他元字符
\<xxx       取单词的头
xxx\>       取单词的尾
\<xx\>      精确匹配
\{n\}       匹配前导符连续出现n次
\{n,\}      匹配前导字符至少出现n次
\{n,m\}     匹配前导字符出现n到m次
\(xxx\)     保存被匹配的字符，作为一个整体
\d          匹配数字grep -P  '\d'  =      grep '[0-9]'
\w          匹配字母数字下划线grep -P  [a-zA-Z0-9]
\s          匹配空格、制表符、换行符        [\t\r\n]

创建JOB

发表于 2020-04-13 | 分类于 oracle

Oracle之JOB学习

今天在工作中创建了一个JOB来定期清洗文件，总结下使用过程

Oracle提供了两个包可以用来创建并管理JOB，分别是DBMS_JOB和DBMS_SCHEDULER,目前官方文档推荐使用DBMS_SCHEDULER，其功能更多

1. DBMS_JOB

1) 创建JOB

语法：

declare
 jobno binary_integer;

  begin
   dbms_job.submit(job=>jobno,
   what=>'xxxx;',
   next_date=>xxxx,
   interval=>'xxxx');
   commit;
end;

示例：

VARIABLE jobno NUMBER;
BEGIN
DBMS_JOB.SUBMIT(
    :jobno,                 ---定义job编号
     'INSERT INTO employees VALUES (7935, ''SALLY'',
     ''DOGAN'', ''sally.dogan@xyzcorp.com'', NULL, SYSDATE,          ''AD_PRES'',  NULL, 
     NULL, NULL, NULL);',   ---执行内容
     SYSDATE,               ---下次执行时间
     'SYSDATE+1'            ---执行间隔
     );
COMMIT;
END;
/
2) 修改JOB
BEGIN
DBMS_JOB.WHAT(
        31, 
        'INSERT INTO employees VALUES (7935, ''TOM'', ''DOGAN'', 
         ''tom.dogan@xyzcorp.com'', NULL, SYSDATE,''AD_PRES'', NULL,
         NULL, NULL, NULL);'    ---修改插入的内容
         );
COMMIT;
END;
/
3) 删除JOB
BEGIN
DBMS_JOB.REMOVE(14144);     ---传参为jobno
COMMIT;
END;
/

2. DBMS_SCHEDULER

1) 创建JOB

语法：

BEGIN
DBMS_SCHEDULER.CREATE_JOB(
   job_name          =>  'job1',
   job_type          =>  'PLSQL_BLOCK',
   job_action        =>  'INSERT INTO employees VALUES (7935, ''SALLY'',
     ''DOGAN'', ''sally.dogan@xyzcorp.com'', NULL, SYSDATE,''AD_PRES'', NULL,
      NULL, NULL, NULL);');
   start_date        =>  SYSDATE,
   repeat_interval   =>  'FREQ = DAILY; INTERVAL = 1');
END;
/
2) 修改JOB
BEGIN
DBMS_SCHEDULER.SET_ATTRIBUTE(
   name          => 'JOB1',
   attribute     => 'job_action',
   value         => 'INSERT INTO employees VALUES (7935, ''TOM'', ''DOGAN'', 
      ''tom.dogan@xyzcorp.com'', NULL, SYSDATE, ''AD_PRES'', NULL,
      NULL, NULL, NULL);');
END;
/
3) 删除JOB
BEGIN
   DBMS_SCHEDULER.DROP_JOB('myjob1');
END;
/
上述内容着重点是对比DBMS_JOB和DBMS_SCHEDULER在创建、修改、删除JOB时的区别，下面记录下今天写的清洗JOB的过程

三种用法：

  1. 自动执行程序:
    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name          =>  'my_new_job1',
program_name      =>  'my_saved_program', 
repeat_interval   =>  'FREQ=DAILY;BYHOUR=12',
comments          =>  'Daily at noon');
END;
/
  2. 自动执行计划
    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name                 =>  'my_new_job2', 
job_type                 =>  'PLSQL_BLOCK',
job_action               =>  'BEGIN SALES_PKG.UPDATE_SALES_SUMMARY; END;',
schedule_name            =>  'my_saved_schedule');
END;
/
  3. 自动执行程序和计划
    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name            =>  'my_new_job3', 
program_name        =>  'my_saved_program1', 
schedule_name       =>  'my_saved_schedule1');
END;
/

    案例

    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name          =>  'TRAN_LAWFILES_JOB',
program_name      =>  'PRO_TRAN_LAWFILES', 
repeat_interval   =>  'FREQ=MINUTELY;INTERVAL=5',
comments          =>  '定时清洗资讯中心文件内容');
END;

    上面语句执行没问题，但是ENABLED=false，auto_drop=true

    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name          =>  'TRAN_LAWFILES_JOB',
program_name      =>  'PRO_TRAN_LAWFILES', 
repeat_interval   =>  'FREQ=MINUTELY;INTERVAL=5',
comments          =>  '定时清洗资讯中心文件内容',
AUTO_DROP         => FALSE,
ENABLED           => TRUE);
END;

    执行报错“PRO_TRAN_LAWFILES必须为program”，没搞懂为什么
    改成：

    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name           =>  'TRAN_LAWFILES_JOB',
job_type           =>  'PLSQL_BLOCK',
job_action         =>  'DECLARE O_NOTE VARCHAR2(200);O_CODE NUMBER(2);BEGIN PRO_TRAN_LAWFILES(O_CODE,O_NOTE);END;',
repeat_interval    =>  'FREQ=MINUTELY;INTERVAL=5', 
auto_drop         => false,
ENABLED            =>  true,
comments           =>  '定时清洗资讯中心文件内容');
END;

    创建成功，查询创建的信息

    select * from user_SCHEDULER_JOBS

    启动job

    begin
 dbms_scheduler.run_job(job_name=>'jobname');
end;
/

    停止job

    BEGIN
DBMS_SCHEDULER.STOP_JOB('job1, sys.dw_jobs');
END;
/

小结

DBMS_SCHEDULER对JOB的管理内容更加丰富，比较全面和细致，而DBMS_JOB的管理属性就要少很多，而且从官方的文档介绍来看，都不愿意过多介绍DBMS_JOB，要么就是内容少，要么就是官方都不愿意多提。

#!/bin/bash
#打印菜单,'<<-'表示后面的END不用靠墙

    h    显示命令帮助
    f    显示磁盘分区
    d    显示磁盘挂载
    m    查看内存使用
    u    查询系统负载
    q    退出程序
    END
while true
do
#用户选择菜单
read -p "请选择要操作的菜单（help h）:" action
case $action in
h|help)
    cat <<-END
        h    显示命令帮助
        f    显示磁盘分区
        d    显示磁盘挂载
        m    查看内存使用
        u    查询系统负载
        q    退出程序
        END
    ;;
f)
    lsblk
    ;;
d)
    df -h
    ;;
m)
    free -m
    ;;
u)
    uptime
    ;;
q)
    exit
    ;;
esac
done

  2. 设置分类、首页、标签等页签

    新生成的博客项目是没有这些页签内容的，点击后会报错：

    首先在博客根目录下创建栏目，比如创建分类

    hexo new categories

    在source底下就新建了一个categories目录了，修改该目录下的index.md，这个文件其实就是博客页面上点击分类页签要显示的页面

    title: 分类
date: 2020-04-08 21:46:29
type: "categories"

    在写的博客中添加categories属性，并指定分类值

    categories: hexo

    此时，再次点击分类就能看到分类情况了
