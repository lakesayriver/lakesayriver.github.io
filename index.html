创建JOB

发表于 2020-04-13 | 分类于 oracle

Oracle之JOB学习

今天在工作中创建了一个JOB来定期清洗文件，总结下使用过程

Oracle提供了两个包可以用来创建并管理JOB，分别是DBMS_JOB和DBMS_SCHEDULER,目前官方文档推荐使用DBMS_SCHEDULER，其功能更多

1. DBMS_JOB

1) 创建JOB

语法：

declare
 jobno binary_integer;

  begin
   dbms_job.submit(job=>jobno,
   what=>'xxxx;',
   next_date=>xxxx,
   interval=>'xxxx');
   commit;
end;

示例：

VARIABLE jobno NUMBER;
BEGIN
DBMS_JOB.SUBMIT(
    :jobno,                 ---定义job编号
     'INSERT INTO employees VALUES (7935, ''SALLY'',
     ''DOGAN'', ''sally.dogan@xyzcorp.com'', NULL, SYSDATE,          ''AD_PRES'',  NULL, 
     NULL, NULL, NULL);',   ---执行内容
     SYSDATE,               ---下次执行时间
     'SYSDATE+1'            ---执行间隔
     );
COMMIT;
END;
/
2) 修改JOB
BEGIN
DBMS_JOB.WHAT(
        31, 
        'INSERT INTO employees VALUES (7935, ''TOM'', ''DOGAN'', 
         ''tom.dogan@xyzcorp.com'', NULL, SYSDATE,''AD_PRES'', NULL,
         NULL, NULL, NULL);'    ---修改插入的内容
         );
COMMIT;
END;
/
3) 删除JOB
BEGIN
DBMS_JOB.REMOVE(14144);     ---传参为jobno
COMMIT;
END;
/

2. DBMS_SCHEDULER

1) 创建JOB

语法：

BEGIN
DBMS_SCHEDULER.CREATE_JOB(
   job_name          =>  'job1',
   job_type          =>  'PLSQL_BLOCK',
   job_action        =>  'INSERT INTO employees VALUES (7935, ''SALLY'',
     ''DOGAN'', ''sally.dogan@xyzcorp.com'', NULL, SYSDATE,''AD_PRES'', NULL,
      NULL, NULL, NULL);');
   start_date        =>  SYSDATE,
   repeat_interval   =>  'FREQ = DAILY; INTERVAL = 1');
END;
/
2) 修改JOB
BEGIN
DBMS_SCHEDULER.SET_ATTRIBUTE(
   name          => 'JOB1',
   attribute     => 'job_action',
   value         => 'INSERT INTO employees VALUES (7935, ''TOM'', ''DOGAN'', 
      ''tom.dogan@xyzcorp.com'', NULL, SYSDATE, ''AD_PRES'', NULL,
      NULL, NULL, NULL);');
END;
/
3) 删除JOB
BEGIN
   DBMS_SCHEDULER.DROP_JOB('myjob1');
END;
/
上述内容着重点是对比DBMS_JOB和DBMS_SCHEDULER在创建、修改、删除JOB时的区别，下面记录下今天写的清洗JOB的过程

三种用法：

  1. 自动执行程序:
    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name          =>  'my_new_job1',
program_name      =>  'my_saved_program', 
repeat_interval   =>  'FREQ=DAILY;BYHOUR=12',
comments          =>  'Daily at noon');
END;
/
  2. 自动执行计划
    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name                 =>  'my_new_job2', 
job_type                 =>  'PLSQL_BLOCK',
job_action               =>  'BEGIN SALES_PKG.UPDATE_SALES_SUMMARY; END;',
schedule_name            =>  'my_saved_schedule');
END;
/
  3. 自动执行程序和计划
    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name            =>  'my_new_job3', 
program_name        =>  'my_saved_program1', 
schedule_name       =>  'my_saved_schedule1');
END;
/

    案例

    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name          =>  'TRAN_LAWFILES_JOB',
program_name      =>  'PRO_TRAN_LAWFILES', 
repeat_interval   =>  'FREQ=MINUTELY;INTERVAL=5',
comments          =>  '定时清洗资讯中心文件内容');
END;

    上面语句执行没问题，但是ENABLED=false，auto_drop=true

    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name          =>  'TRAN_LAWFILES_JOB',
program_name      =>  'PRO_TRAN_LAWFILES', 
repeat_interval   =>  'FREQ=MINUTELY;INTERVAL=5',
comments          =>  '定时清洗资讯中心文件内容',
AUTO_DROP         => FALSE,
ENABLED           => TRUE);
END;

    执行报错“PRO_TRAN_LAWFILES必须为program”，没搞懂为什么
    改成：

    BEGIN
DBMS_SCHEDULER.CREATE_JOB (
job_name           =>  'TRAN_LAWFILES_JOB',
job_type           =>  'PLSQL_BLOCK',
job_action         =>  'DECLARE O_NOTE VARCHAR2(200);O_CODE NUMBER(2);BEGIN PRO_TRAN_LAWFILES(O_CODE,O_NOTE);END;',
repeat_interval    =>  'FREQ=MINUTELY;INTERVAL=5', 
auto_drop         => false,
ENABLED            =>  true,
comments           =>  '定时清洗资讯中心文件内容');
END;

    创建成功，查询创建的信息

    select * from user_SCHEDULER_JOBS

    启动job

    begin
 dbms_scheduler.run_job(job_name=>'jobname');
end;
/

    停止job

    BEGIN
DBMS_SCHEDULER.STOP_JOB('job1, sys.dw_jobs');
END;
/

小结

DBMS_SCHEDULER对JOB的管理内容更加丰富，比较全面和细致，而DBMS_JOB的管理属性就要少很多，而且从官方的文档介绍来看，都不愿意过多介绍DBMS_JOB，要么就是内容少，要么就是官方都不愿意多提。

    h    显示命令帮助

    d    显示磁盘挂载

    q    退出程序

#用户选择菜单

h|help)
    cat <<-END
        h    显示命令帮助

        d    显示磁盘挂载
        m    查看内存使用

    ;;

    ;;

    ;;

    uptime
    ;;
q)
    exit
    ;;
esac
done

    一次赋多个值

    数组名=([下标/索引]=变量1 [下标/索引]=变量2)

    其他赋值方式，通过运算得到

    let 数组名[下标或索引]++

    其他变量定义

  3. 取出一个目录下的目录和文件：dirname 目录 和 basename 目录
  4. 变量内容的删除和替换

    %表示从右往左去掉一个/key/
    %%表示从右往左去掉最大长度
    “#”表示从左往右去掉一个
    “##”表示从左往右去掉最大

    url=lakesayriver.github.io
echo ${#url}    #url的长度
echo ${url#.}    #.表示标志位
github.io
echo ${url##.}
io
echo ${url%.}
lakesayriver.github

    统计web服务的不同状态的连接个数

    查看网站连接状态的命令 ss -natp|grep :80
    循环计算不同状态 

    #!/bin/bash
declare -A array
state=`ss -natp|grep :80|cut -d' ' -f1`
for i in $state
do
 let array[$i]++
done
#打印各个状态的连接数
for j in ${!array[@]}
do
 echo $j:$array[$j]
done

RANDOM

随机产生0-32767之前的数

案例1：

产生一个phonenum文件，随机产生以139开头的手机号1000个，每个一行

sort -u 去重
let i++ 变量可以不加$

#!/bin/bash
file=./phonenum.txt
i=0
while [ $i -lt 1000 ]
do
    j=0
    phone=139
    while[ $j -lt 8 ]
    do
        phone=$phone$[$RANDOM%10]
        let j++
    done
    echo $phone >> file
    let i++
done
案例2：

在上面生成的1000个手机号中随机抽取五个幸运观众,显示前三位和后四位

wc -l filename 打印文件行号和内容
a=12345678;${a:3:4} 获取变量a的第四位开始取4位数字
sed -i 直接在文件上编辑
sed -i ‘n,md’ filename 删除filename文件的第n到m行
sed -i “${var}d”
把执行命令赋值给变量 变量=cmd

脚本代码

#!/bin/bash 
file=./phonenum.txt
i=0
whie [ $i -le 4 ]
do
 line=`wc -l $file|cut -d' ' -f1`
 luck_line=$[$RANDOM%$line+1]
 luck_one=`head -$luck_line $file|tail -1`
 #显示到屏幕
 echo "139****${luck_one:7:4}"
 #删除抽中的号码
 sed -i "/$luck_line/d" file
 let i++
done
